# 创新思维框架：打破局限操作手册

> 用途：在科研的"方法设计"阶段，系统性地识别创新空间。

---

## 核心认知：两类约束

```
约束类型     │ 是否可打破  │ 判断标准
─────────────┼────────────┼────────────────────────────────────
物理铁律     │    ❌       │ 热力学、守恒定律、数学定理
实验伦理     │    ❌       │ 人体安全、数据隐私、学术诚信
统计必要条件 │    ❌       │ 足够样本量、避免循环论证
─────────────┼────────────┼────────────────────────────────────
领域惯例     │    ✅       │ "大家都这么做"但无理论必要性
默认假设     │    ✅       │ 线性、平稳、独立同分布等
评价习惯     │    ✅       │ 某个指标成为"标准"但不一定最优
模型架构     │    ✅       │ CNN/Transformer 不是唯一选择
问题定义     │    ✅       │ 甚至可以重新定义要解决的问题
```

## 五步创新识别法

### Step 1：列出领域假设清单

针对当前研究主题，列出所有隐含假设：

```
领域：[填入研究领域]

默认假设：
□ 数据假设：...（如"样本独立同分布"）
□ 模型假设：...（如"线性关系"）
□ 评估假设：...（如"MSE 是最优损失"）
□ 问题假设：...（如"这是一个回归问题"）
□ 数据量假设：...（如"需要大量标注数据"）
```

### Step 2：逐一质疑假设

对每个假设问：

1. **这个假设真的成立吗？** — 找反例或反证
2. **谁率先质疑了它？** — 联网搜索相关"挑战传统"的论文
3. **打破它会带来什么？** — 评估收益与代价

### Step 3：创新方向矩阵

```
维度         │ 现有方案          │ 可创新方向
─────────────┼──────────────────┼───────────────────────────
数据层       │ 全量监督学习      │ 弱监督/自监督/小样本
特征层       │ 手工特征          │ 自动特征发现/物理约束嵌入
模型层       │ 通用架构          │ 领域特化归纳偏置
优化层       │ 标准梯度下降      │ 元学习/多任务/课程学习
评估层       │ 单指标            │ 多维度/应用场景导向指标
问题定义层   │ 现有任务定义      │ 重新定义问题边界
```

### Step 4：联网搜索验证创新性

对每个创新思路执行：

```bash
搜索关键词：
  "[创新方向] [领域]"
  "[创新方向] recent survey"
  "[创新方向] benchmark"

目标：
  - 确认是否已有人做过
  - 若有：找到最新进展，在其基础上进一步创新
  - 若无：评估可行性和潜在价值
```

### Step 5：创新点可行性评估

```
创新点：[描述]
─────────────────────────────────────
新颖性：⭐⭐⭐⭐⭐（1-5）
可行性：⭐⭐⭐⭐⭐（1-5）
影响力：⭐⭐⭐⭐⭐（1-5）
资源需求：[数据量/算力/时间]
主要风险：[列出 1-3 个]
─────────────────────────────────────
推荐理由：...
```

## 常见创新思路库（按领域）

### 时间序列 / 预测

- 引入物理先验（微分方程约束）
- 非稳态建模（不假设平稳性）
- 事件驱动建模（而非等间隔采样）
- 不确定性量化（预测区间而非点估计）

### 机器学习 / 深度学习

- 打破独立同分布假设（图结构/因果结构）
- 引入领域知识作为归纳偏置
- 样本效率优化（弱监督/数据增强）
- 模型可解释性与性能的新权衡

### 自然语言处理

- 非文本模态融合
- 长尾知识强化
- 符号推理与神经网络结合

### 生命科学 / 材料 / 化学

- 多尺度建模
- 实验 + 仿真协同
- 反事实推理（"如果改变这个条件..."）

## 创新边界提醒

```
⚠️ 常见"伪创新"陷阱

1. 换个数据集就叫创新  → 不算，除非数据集有本质新颖性
2. 把两个现有方法直接拼接 → 需要说明为什么1+1>2
3. 调参得到更好结果 → 需要可复现，且调参方法本身要公正
4. 在特定数据集上过拟合 → 不构成泛化性创新
5. 只改论文写法，方法无实质变化 → 学术不端
```
