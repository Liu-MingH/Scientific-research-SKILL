# 因果推断检查清单

> 用途：在每个实验节点完成后，逐项核查，防止虚假结论进入下一阶段。

---

## 第一层：数据可靠性

| 检查项 | 问题 | 通过标准 |
|--------|------|----------|
| 数据源 | 数据来自哪里？是否可信？ | 有明确来源，官方或公开数据集 |
| 数据完整性 | 缺失值如何处理？ | 记录缺失比例，处理方法合理 |
| 标签质量 | 标签/目标变量是否可信？ | 无明显噪声/错误标注 |
| 时间顺序 | 时序数据是否保持正确顺序？ | 训练集时间早于测试集时间 |
| 数据泄露 | 测试集信息是否渗漏到训练集？ | 严格隔离，无特征工程泄露 |

---

## 第二层：实验设计可靠性

| 检查项 | 问题 | 通过标准 |
|--------|------|----------|
| 基线公平性 | 基线方法是否在相同数据/条件下测试？ | 统一数据划分、随机种子 |
| 超参数公平性 | 是否只对自己方法调参？ | 基线也要适当调参 |
| 多次运行 | 结果是否只跑了一次？ | 建议至少 3-5 次，报告均值±标准差 |
| 种子固定 | 随机种子是否记录？ | 所有随机操作固定并记录种子 |
| 评估指标 | 指标是否真正反映目标？ | 指标选择有理论依据 |

---

## 第三层：因果性 vs 相关性

```
核心问题：我观察到的结果，是"因为我的方法更好"，
         还是"因为某个混淆变量"？
```

### 常见混淆因子

1. **过拟合** — 在训练集/验证集上好，测试集上差
   - 检查：训练/验证/测试三套指标是否一致
   
2. **数据分布偏移** — 测试集和训练集来自不同分布
   - 检查：比较两者的统计特征（均值、方差、分布形状）

3. **评价指标不当** — 用 MSE 衡量含异常值的序列
   - 检查：用多个互补指标交叉验证

4. **基线实现错误** — 基线代码存在 Bug 导致低估
   - 检查：在标准公开数据集上复现基线的文献结果

5. **超参数偷跑** — 用测试集信息选择超参数
   - 检查：超参数选择只能基于验证集

6. **选择性汇报** — 只汇报好的 run，隐藏差的 run
   - 要求：汇报全部 run 的统计分布

## 第四层：异常结果处理规程

```
当结果"好得出奇"（超出领域 SOTA 20% 以上）时：

第一反应：怀疑 Bug，不是庆祝

检查顺序：
1. 评估函数是否实现正确？
2. 数据划分是否有泄露？
3. 标签是否被错误使用？
4. 基线实现是否存在缺陷？
5. 如果以上都没问题，才考虑是真实提升
   → 此时必须用额外独立数据集验证
```

---

## 第五层：结论陈述规范

- ✅ 正确："在本实验设置下，方法 A 在数据集 X 上比基线 B 降低了 MAE 15.3%（±2.1%，n=5次运行）"
- ❌ 错误："方法 A 全面优于现有方法"（过度泛化）
- ❌ 错误："证明了 A 导致了 B"（未满足因果推断条件就用因果语言）
- ✅ 正确："实验结果表明 A 与 B 存在强相关，但因果关系需要控制变量实验进一步验证"
